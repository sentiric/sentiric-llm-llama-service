# 🧠 AI SYSTEM INSTRUCTIONS (SOP-00 COMPLIANCE)

Bu repoda kod yazan bir insan veya Yapay Zeka Ajanı (Cursor, Copilot vb.) AŞAĞIDAKİ KURALLARA KESİNLİKLE UYMAK ZORUNDADIR:

## 1. KÜTÜPHANE VE API SINIRLARI
* **Motor:** `llama.cpp`'nin `b7415` sürümüne sıkı sıkıya bağlıyız. 
* **KV Cache API (KRİTİK):** Eski `llama_kv_cache_seq_rm` fonksiyonu YASAKTIR. Tüm KV Cache manipülasyonları `llama_memory_t mem = llama_get_memory(ctx);` üzerinden `llama_memory_seq_rm(mem, ...)` ile yapılmalıdır.
* **Tokenizasyon:** `llama_tokenize` fonksiyonu artık `llama_context` değil, `llama_vocab*` parametresi alır.

## 2. KESİN YASAKLAR (DO NOTS)
* **Durum (State) Yönetimi:** Servis %100 Stateless olmalıdır. Kullanıcı veya SAGA geçmişini kalıcı veritabanına yazamaz.
* **Standart Log Yasaktır:** `std::cout`, `printf` veya saf `spdlog` çağrıları YASAKTIR. Sadece `suts_logger.h` içindeki `SUTS_INFO`, `SUTS_ERROR` makroları kullanılmalıdır. 
* **Metadata Hatası:** Rust'tan gelen `trace_id` değerleri `Some("uuid")` formatında gelebilir. Bu değerler doğrudan loglanamaz, gRPC metadata üzerinden saf string olarak çekilip kontrol edilmelidir. `tenant_id` zorunludur!

## 3. BELLEK YÖNETİMİ (VRAM SAFETY)
* `LlamaContextPool` içindeki `std::mutex` ve `std::condition_variable` kilit yapıları ASLA esnetilemez veya by-pass edilemez. Multithreading yarış durumları (Race Condition) VRAM OOM (Out-of-Memory) hatalarına yol açar.
